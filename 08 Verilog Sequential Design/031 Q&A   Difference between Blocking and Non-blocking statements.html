Here is how to differentiate between where/when to use blocking/non-blocking assignments in your design.


SEQ1: Blocking procedural assignment. Expression is evaluated and assigned when the statement is encountered. In a begin—end sequential statement sequence, execution of the next statement is blocked until the current assignment is complete.

begin // SEQ1
    a=b;
    b=a;
end

In SEQ1 the first assignment changes a before the second assignment reads a.


SEQ2: Non-blocking procedural assignment. The expression is evaluated when the statement is encountered, and assignment is postponed until the end of the simulation time-step. In a begin—end sequential statement sequence, the execution

of the next statement is not blocked; and will be evaluated before the assignment is complete.

begin // SEQ2
    a<=b;
    b<=a;
end

In SEQ2 both assignments will be evaluated before a or b changes.


MODELING TIP (industry recommendation) for best synthesis results with most tools: To avoid potential simulation race conditions in zero-delay models:

• Use blocking assignments ( = ) to model combinational logic.

• Use non-blocking assignments ( <= ) to model sequential logic.

always @(*) begin // combinational
    variable = expression;
end


always @(posedge clk) begin // sequential
    variable <= expression;
end


Please note that if you don't respect the above rules you may have a whole universe of errors that are very hard to debug.
