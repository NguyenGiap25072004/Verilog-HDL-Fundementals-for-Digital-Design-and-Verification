I created the second variant of the code exactly to emphasize this concept that you asked about.


When I said "if qa is set in clock cycle 'N' qb is set in 'N+1'" I was referring at the fact that the non-blocking operator "<=" executes in parallel those two lines of code whenever a 'posedge clk' occurs.


Whenever a posedge clock occurs then the following happens in parallel:

1. qa updates with the value from the right-hand side ~qa & ~qb (Note the values of qa and qb before the posedge)

2. qb copies the value of qa that was exactly before the posedge (the old value of a)


This can be easily seen from the waveform also.
